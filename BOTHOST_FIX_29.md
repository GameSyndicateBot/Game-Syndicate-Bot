# Bothost fix 29

Изменения:
- добавлен HTTP health-сервер на `0.0.0.0:${PORT:-3000}`;
- в Dockerfile добавлены `PORT=3000`, `EXPOSE 3000` и Docker `HEALTHCHECK`;
- Discord-бот продолжает запускаться через прежний `index.js`;
- при остановке контейнера health-сервер корректно закрывается.

## Переменные Bothost
Обязательно:
- `TOKEN`
- `CLIENT_ID`
- `GUILD_ID`

Рекомендуется:
- `PORT=3000`
- `DATABASE_PATH=/app/shared/database.sqlite`
- `BACKUP_DIR=/app/shared/backups`

После загрузки файлов в корень GitHub-репозитория выполните полную пересборку контейнера, а не только перезапуск.

Важно: сообщение панели «не удалось получить URL агента» относится к внутреннему агенту логов Bothost. Health endpoint добавлен для совместимости с проверкой контейнера, однако если сама панель продолжит показывать эту фразу при зелёном статусе «Работает», это неисправность лог-сервиса Bothost, а не ошибка Node.js-кода.
