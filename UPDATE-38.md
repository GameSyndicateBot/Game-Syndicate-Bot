# GS BUILD 38 — Production Startup Hardening

- Добавлен runtime-маркер сборки непосредственно в `index.js`; он отображается независимо от Docker/startup-команды платформы.
- Добавлена process-wide защита `clientReady` от повторного запуска всех фоновых систем.
- Логи Game Lobby теперь меняются только при фактическом подключении Discord или Telegram, а не при повторной передаче ссылок на клиенты.
- Единственный таймер автозакрытия Game Lobby по-прежнему защищён process-wide singleton.
- Игровая логика, награды и структура базы не изменены.
