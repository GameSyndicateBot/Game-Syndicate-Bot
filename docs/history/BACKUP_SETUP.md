# Надёжные бэкапы Game Syndicate

Патч делает согласованный SQLite-снимок через `better-sqlite3 backup()` и каждые 6 часов отправляет его в приватный Discord-канал. Это защищает прогресс даже тогда, когда Bothost пересоздаёт контейнер и удаляет его файловую систему.

## Обязательная настройка

1. Создай приватный текстовый канал, например `🔒・бэкапы-бота`.
2. Разреши доступ только себе и боту.
3. Скопируй ID канала и добавь в Bothost:

```env
BACKUP_CHANNEL_ID=ID_КАНАЛА
BACKUP_INTERVAL_MINUTES=360
BACKUP_RETENTION=30
```

4. Если Bothost предоставляет постоянный диск, примонтируй его в `/data` и используй:

```env
DATABASE_PATH=/data/database.sqlite
BACKUP_DIR=/data/backups
```

Если постоянного диска нет, можно временно оставить `/app/database/...`, но Discord-бэкапы становятся обязательными.

## Что происходит автоматически

- через 30 секунд после запуска создаётся startup-бэкап;
- затем бэкап создаётся каждые 6 часов;
- файл загружается в указанный Discord-канал;
- локально сохраняются последние 30 файлов;
- `/backup` продолжает создавать и отдавать ручной снимок.

## Восстановление

1. Останови бота.
2. Скачай последний `.sqlite` из приватного канала.
3. Назови его `database.sqlite`.
4. Помести в постоянное хранилище по пути из `DATABASE_PATH`.
5. Запусти бота.

Не добавляй рабочую базу и бэкапы в публичный GitHub: они содержат IDs и весь прогресс участников.
